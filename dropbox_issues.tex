\begin{enumerate}
	\item Bad at dealing with simultaneous changes: ``I never make changes, edit or work with files in dropbox. Thatâ€™s a very bad idea. Dropbox is very poor at dealing with simultaneous changes to a document.'' (\url{http://www.whooshtranscription.com/dropbox-review/})
		\begin{itemize}
			\item Conflicted copies. These happen any time two people update a file at the same time. Or one person updates a file on a second device when the first has not synced. i.e. ``at the same time'' only means that a sync has not occurred. (\url{http://nerdenterprises.com/blog/2011/05/how-to-avoid-conflicting-copies-in-drop-box/})
			\begin{itemize}
				\item Procedure for creating conflicted copy with oneself
					\begin{enumerate}
						\item Create file and wait for it to sync to Dropbox and then down to second device
						\item Disconnect second device from internet
						\item Make a change to each file (even the exact same change)
						\item Reconnect second device to the internet. Wait for sync to occur
						\item Voil\`{a}
					\end{enumerate}
				\item Much easier to create conflicted copy with a collaborator...
			\end{itemize}
		\end{itemize}
	\item Automatic sync does not always work. Not clear how to replicate this. (\url{http://www.whooshtranscription.com/dropbox-review/}). Some possibilities:
		\begin{itemize}
		  \item Dropbox mea culpa about OS incompatibilities (\url{https://www.dropbox.com/en/help/145})
			\item Confusion about two different types of ``sharing;'' one only sends the files once; the other sends all future updates as well \\ (\url{http://simonhackett.com/2013/04/05/dropbox-sharing-doesnt-work-and-the-fi/})
		\end{itemize}
	\item 30-day retrieval: ``For a free account any file deleted in the in the last 30 days can be undeleted.'' (\url{http://www.whooshtranscription.com/dropbox-review/})
		\begin{itemize}
			\item If you have Dropbox Pro, you can purchase Extended Version History for $\$$39/yr to get 365-day retrieval
			\item Packrat was the same price with unlimited version history, but has been phased out
		\end{itemize}
	\item Files are deleted if anyone sharing them deletes them or moves them out of dropbox and disconnects the new location \\
	(\url{http://www.businessinsider.com/weird-feature-in-dropbox-deletes-files-2014-4})	
	\item Out of control directories with multiple date-stamped versions, futile attempts to use text files to keep records of what you've
done.
\end{enumerate}
